# SwapU 校园闲置物品交易平台 - 需求分析文档

## 1. 引言

### 1.1 项目背景
高校学生群体中，教材、电子产品、生活用品等闲置物品资源丰富，但往往因为毕业、课程结束等原因被闲置或丢弃，造成资源浪费。低年级学生或其他有需求的学生希望能以低价获取这些物品。然而，传统的线下跳蚤市场受时间和场地限制，社交群组（如微信群、QQ群）发布信息碎片化、检索困难且缺乏信任机制。

### 1.2 项目目的
本项目旨在开发一个基于 Vue3 + SpringBoot 的校园专属闲置物品交易平台 "SwapU"。通过构建前后端分离的系统，提供便捷、安全、高效的闲置物品流通渠道，解决信息不对称问题，促进校园绿色循环经济的发展。

### 1.3 适用范围
本系统主要面向高校在校学生（作为买家/卖家/置换者）以及校园管理员。

---

## 2. 用户角色

系统包含两类主要角色：

1.  **普通学生用户**：
    -   拥有校园身份认证（如学号/教务系统认证）。
    -   既可以是物品发布者（卖家），也可以是物品需求者（买家）。
    -   可以进行物品发布、浏览、搜索、购买、置换、评价等操作。

2.  **系统管理员**：
    -   负责平台的日常运营和监管。
    -   负责用户审核、物品审核、投诉处理、系统配置及数据统计。

## 3. 功能需求

### 3.1 用户管理模块（已完成）
*   **注册与登录**：
    -   支持学号/邮箱注册，基于 JWT 的身份认证流程。
    -   支持密码登录及找回密码功能。
    -   登录成功后返回 Token，前端自动存储并携带于后续请求头。
*   **个人信息管理**：
    -   用户可编辑头像、昵称、联系方式（手机/微信/QQ）。
    -   支持头像上传功能，后端实现文件上传接口。
    -   查看并维护个人收货地址/自提偏好。
    -   查看个人信用评分（基于交易评价生成）。
*   **密码安全**：
    -   独立的安全设置页面，支持修改密码。
    -   强制二次确认新密码，确保密码安全。

### 3.2 物品管理模块（已完成）
*   **物品发布**：
    -   支持上传物品图片（多张），提交时自动转换为 JSON 字符串存储。
    -   填写物品名称、详细描述、价格/原价、分类（教材、电子、生活、美妆等）。
    -   设置新旧程度、交易方式（出售/置换/赠送）、校区信息。
    -   后端自动注入当前登录用户 ID，默认状态设为在售。
*   **物品编辑与管理**：
    -   发布者可修改物品信息。
    -   支持下架（暂停交易）和上架（恢复交易）操作。
    -   支持删除（物理删除物品数据），操作前有确认框提示。
*   **物品浏览与检索**：
    -   **首页展示**：瀑布流布局展示物品卡片，包含封面、价格、标题、发布者信息、新旧程度标签。
    -   **物品详情页**：左侧图片轮播支持大图预览，右侧展示详细信息、卖家卡片、操作按钮区。
    -   **关键词搜索**：支持模糊搜索物品名称、描述，集成 Elasticsearch 实现高效搜索。
    -   **多条件筛选**：按分类、价格区间、新旧程度、发布时间筛选。
    -   **搜索建议**：根据用户输入提供智能搜索建议。
*   **收藏功能**：
    -   用户可以收藏感兴趣的物品，实时切换收藏状态。
    -   详情页自动检查当前用户收藏状态，点击收藏按钮实时更新样式和数量。
    -   个人中心可查看"我收藏的物品"列表。

### 3.3 交易置换模块（已完成）
*   **交易流程**：
    -   **下单**：买家发起购买请求，系统生成订单，并支持选择“线上支付”或“线下自行交易”。
    -   **确认**：线上支付订单需卖家确认接单；线下交易订单需卖家当面确认收款后“同意并完成”。
    -   **支付**：集成支付宝沙箱支付，支持线上支付流程（包含支付回调和通知处理）。线上支付订单若30分钟未付款，通过 RocketMQ 延迟队列自动取消并归还库存。
    -   **完成**：双方确认收货/交付，订单状态流转为"已完成"。
    -   **取消**：支持订单取消操作，可在待确认或未支付状态下取消订单。
*   **订单管理**：
    -   查看"我发布的"（卖出/送出）和"我买到的"（买入/收到）订单列表。
    -   订单状态跟踪：待接单、待支付、进行中/待交付、已完成、已取消。
    -   订单详情页展示完整的交易信息，包括物品信息、买卖双方信息、交易金额、支付方式、状态等。
*   **交易方式**：
    -   支持自提、送货上门等校园场景适用的面交方式。

### 3.4 评价与投诉模块（已完成）
*   **双向评价**：
    -   交易完成后，买卖双方可互评（评分+文字评论+图片）。
    -   评价内容公开展示在用户主页及物品详情页（作为信用参考）。
    -   支持查看订单相关的评价、物品评价列表、用户收到的评价列表。
*   **投诉与举报**：
    -   用户可举报违规物品（如虚假信息、违禁品）或交易纠纷。
    -   支持上传图片证据（JSON 格式存储）。
    -   举报状态跟踪：待处理、已处理、已驳回。
    -   管理员可查看举报详情并进行处理。

### 3.5 互动与通知模块（已完成）
*   **商品留言板**：
    -   商品详情页底部提供留言功能，支持买家针对商品公开提问（如“能刀吗？”）。
    -   支持多层级回复（卖家及其他用户可参与互动）。
*   **实时聊天 (私信)**：
    -   集成 WebSocket，支持买卖双方实时点对点私聊。
    -   支持发送文本消息、图片等，具备断线重连和心跳保活机制。
    -   系统顶部导航栏展示未读消息全局红点提醒。
    -   支持已读标记及最近联系人列表漫游。
*   **系统公告**：
    -   管理员可发布系统公告，支持草稿和发布状态。
    -   用户可查看公告列表和公告详情。
*   **站内通知**：
    -   支持多种通知类型：系统通知、交易通知、互动通知（如新留言、新订单等）。
    -   未读消息计数功能，实时显示未读通知数量。
    -   支持标记单条通知为已读、一键标记所有通知为已读。
    -   通知关联相关业务 ID，可跳转到对应业务页面。

### 3.6 AI 智能助手模块（已完成）
*   **智能商品推荐**：
    -   基于 RAG（检索增强生成）技术，使用 PgVector 向量数据库存储商品信息。
    -   用户每成功发布一个商品，将其标题和描述作为整体存入向量数据库。
    -   用户输入查询词后，系统自动检索最相关的商品（Top 3）。
*   **AI 对话**：
    -   集成 DeepSeek 大模型，提供智能对话能力。
    -   支持多轮对话，保留最近对话历史（6条/3轮）。
    -   根据用户提问智能推荐商品，或进行日常闲聊。
    -   推荐结果以商品卡片形式展示，包含标题、价格、描述等信息。

### 3.7 管理员后台模块（已完成）
*   **用户管理**：
    -   查看用户列表，支持分页查询。
    -   修改用户状态（正常/冻结），冻结违规用户账号。
    -   查看用户详情信息。
*   **物品审核**：
    -   查看物品列表，支持分页查询。
    -   审核物品状态：待审核、在售、交易中、已售出、已下架。
    -   强制下架违规物品。
    -   支持 Elasticsearch 数据同步操作。
*   **分类管理**：
    -   管理物品分类标签，支持增删改查。
    -   分类信息包含名称、排序、状态等。
*   **举报处理**：
    -   查看举报列表，支持分页查询。
    -   查看举报详情，包括举报人、被举报对象、类型、原因、证据图片等。
    -   处理举报，填写处理结果，状态更新为已处理或已驳回。
*   **公告管理**：
    -   发布、编辑、删除系统公告。
    -   支持草稿和发布状态管理。
*   **数据统计**：
    -   可视化展示平台数据仪表盘。
    -   统计数据包括：总用户数、今日新增用户、总物品数、今日新增物品、总订单数、今日新增订单、待处理举报数等。
    -   热门搜索词统计，了解用户搜索趋势。

## 4. 非功能需求

### 4.1 安全性
*   **身份验证**：采用 JWT (JSON Web Token) 进行用户鉴权。
*   **数据加密**：用户密码加密存储（BCrypt），敏感信息（如联系方式）在非交易状态下脱敏展示。
*   **权限控制**：严格区分普通用户与管理员权限，防止越权操作。
    -   普通用户拦截器（LoginInterceptor）验证登录状态。
    -   管理员拦截器（AdminInterceptor）验证管理员权限。
*   **防攻击**：
    -   防止 SQL 注入（使用 MyBatis-Plus 参数化查询）。
    -   全局异常处理，统一返回格式，避免敏感信息泄露。

### 4.2 性能
*   **响应时间**：页面加载及 API 响应时间应在 2秒以内。
*   **并发支持**：支持一定量的校园用户并发访问，特别是在开学季等高峰期。
*   **缓存优化**：
    -   Redis 缓存配置，提升系统响应速度。
    -   Redisson 分布式锁支持。
*   **搜索优化**：
    -   Elasticsearch 全文搜索引擎，支持高效商品检索。
    -   向量数据库（PgVector）支持语义相似度搜索。

### 4.3 易用性
*   **界面交互**：Vue3 构建响应式前端，界面简洁美观，适配移动端（手机浏览器/H5）和桌面端。
*   **操作流程**：发布和购买流程尽可能简化，减少点击次数。
*   **UI 设计**：
    -   采用"闲鱼黄"(#ffda44)作为主色调，卡片式布局提升视觉体验。
    -   空状态处理，无数据时引导用户操作。
    -   按钮差异化设计，上架(黄色强调)、下架(灰色弱化)、删除(红框警示)。

### 4.4 可维护性
*   **代码规范**：统一的代码风格和目录结构。
*   **日志记录**：全局日志切面（GlobalLogAspect）记录请求和响应信息。
*   **文档齐全**：
    -   API 接口文档（API_DOC.md）
    -   开发日志（DEVELOPMENT_LOG.md）
    -   部署文档（DEPLOY.md）
    -   阶段划分报告（PHASE_REPORT.md）

## 5. 技术路线与架构

*   **前端**：Vue 3 (Composition API) + Vue Router + Pinia (状态管理) + Element Plus + Vant UI (组件库) + Vite (构建工具)
*   **后端**：Spring Boot 2.7 + MyBatis-Plus + JPA (Spring Data Elasticsearch)
*   **数据库**：
    -   MySQL 8.0 (主数据库)
    -   PgVector (向量数据库，用于 AI 商品推荐)
*   **搜索引擎**：Elasticsearch 7.17 (全文搜索)
*   **缓存**：Redis + Redisson
*   **消息队列**：RocketMQ (物品数据同步)
*   **AI 能力**：
    -   DeepSeek 大模型 (AI 对话)
    -   Spring AI (AI 框架集成)
    -   Ollama + bge-m3 (本地向量模型)
*   **支付**：支付宝 SDK 集成
*   **部署**：Docker + Docker Compose，支持多容器管理

## 6. 数据字典

### 6.1 用户相关表
*   `sys_user` (用户表): user_id, username, password, student_no, real_name, college, phone, wechat, qq, avatar, credit_score, role(0-普通用户,1-管理员), status, created_at, updated_at
*   `search_history` (搜索历史表): id, keyword, count, updated_at

### 6.2 物品相关表
*   `item` (物品表): item_id, user_id, category_id, title, description, price, original_price, images(JSON), condition_rate(新旧程度), transaction_method(交易方式), status(0-待审核,1-在售,2-交易中,3-已售出,4-已下架), view_count, want_count, campus_area, created_at, updated_at
*   `item_category` (分类表): category_id, name, sort_order, status, created_at, updated_at
*   `item_want` (收藏表): want_id, user_id, item_id, created_at

### 6.3 交易相关表
*   `trade_order` (订单表): order_id, order_no, buyer_id, seller_id, item_id, amount, transaction_method, status(0-待确认,1-进行中,2-已完成,3-已取消), payment_method(1-线上支付,2-线下自行交易), payment_status(0-未支付,1-已支付), trade_no(支付宝交易号), pay_time, buyer_note, created_at, updated_at, completed_at

### 6.4 互动与评价表
*   `item_message` (商品留言表): message_id, item_id, user_id, content, parent_id, reply_to_user_id, created_at
*   `chat_message` (私聊消息表): msg_id, sender_id, receiver_id, content, msg_type(1-文本,2-图片), is_read, created_at
*   `comment` (评价表): comment_id, order_id, item_id, user_id, target_user_id, content, rating, images, created_at
*   `report` (举报表): report_id, reporter_id, target_id, type(1-违规物品,2-交易纠纷), reason, images(JSON), status(0-待处理,1-已处理,2-已驳回), result, created_at, updated_at

### 6.5 消息通知表
*   `sys_announcement` (系统公告表): announcement_id, title, content, status(0-草稿,1-发布), created_at, updated_at
*   `sys_notification` (站内通知表): notification_id, user_id, type(1-系统,2-交易,3-互动), title, content, related_id, is_read(0-未读,1-已读), created_at

### 6.6 向量检索相关表
*   `vector_store` (向量存储表): id, content, metadata, embedding (由 Spring AI 自动管理)

## 7. 开发阶段总结

### 7.1 已完成功能（当前最新）
1.  **用户管理模块**：注册登录、JWT 认证、个人信息管理、头像上传、密码修改
2.  **物品管理模块**：物品发布（多图上传）、物品编辑、上下架、删除、瀑布流展示、详情页、Elasticsearch 搜索、多条件筛选、收藏功能
3.  **交易模块**：双规支付流程（线上支付宝/线下当面付）、自动取消订单防锁单机制、订单列表及状态跟踪
4.  **互动与通信模块**：商品详情页留言板、WebSocket 实时私信系统（含红点未读提示）
5.  **评价与举报**：双向评价（评分+文字+图片）、评价列表展示、举报审核流程
6.  **消息通知模块**：系统公告、站内通知
7.  **AI 助手模块**：RAG 商品推荐、DeepSeek 智能对话
8.  **管理员后台**：用户管理、物品审核、分类管理、举报处理、公告管理、数据统计仪表盘

### 7.2 技术亮点
-   前后端分离架构，RESTful API 设计
-   JWT 身份认证 + 拦截器权限控制
-   Elasticsearch 全文搜索 + PgVector 向量搜索双引擎
-   RAG 技术实现智能商品推荐
-   WebSocket 实时通信协议解决私信痛点
-   RocketMQ 延迟消息实现高并发订单防锁单
-   支付宝沙箱支付完整集成
-   Docker Compose 一键部署
-   全局日志切面、统一异常处理
